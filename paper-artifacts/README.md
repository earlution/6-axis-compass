# Paper Artifacts

This directory contains static, versioned artifacts for the *A Common Enemy* academic paper and related research outputs. Every file is generated by `npm run generate-artifacts` and committed to the repository so that LaTeX compilation requires no network access.

## Directory Layout

```
paper-artifacts/
├── actors/                  # One directory entry per political actor
│   ├── Conservative-Party.json
│   ├── Conservative-Party.svg
│   ├── Conservative-Party.tex
│   ├── Labour-Party.json
│   ├── Labour-Party.svg
│   ├── Labour-Party.tex
│   └── ... (27 actors total)
└── comparisons/             # Multi-actor overlay charts
    ├── uk-parties.json
    ├── uk-parties.svg
    ├── uk-vs-us.json
    ├── uk-vs-us.svg
    └── ... (5 groups total)
```

## File Formats

### `.json` — Traceable Data

Contains the raw six-axis scores plus full provenance metadata:

| Field | Meaning |
|-------|---------|
| `source` | Tool name and version |
| `gitCommit` | Full SHA of the commit used to generate the file |
| `generated` | ISO-8601 timestamp |
| `actor` | Display name |
| `slug` | URL-safe identifier |
| `category` | e.g. "UK Political Party" |
| `color` | Hex colour used in charts |
| `axes` | Map of axis name → score (0–10) |
| `traceability` | Map of axis name → `{confidence, rationale, sources[]}` |

Each source in `sources[]` has:
- `type` — `manifesto`, `legislation`, `vote`, `policy`, `speech`, `academic`, `media`, or `other`
- `title` — Short title
- `date` — ISO-8601 date
- `url` — Permalink
- `relevantText` — Quoted excerpt supporting the score
- `citation` — Full academic or parliamentary citation

### `.svg` — Print-Ready Vector Chart

- 600 × 600 px viewBox
- Self-contained, no external stylesheets
- White background with light grey border for print compatibility
- Uses the actor's brand colour for the polygon fill and stroke
- Dark-theme axes and labels converted to `#333333` for legibility on white paper

### `.tex` — LaTeX Command Definitions

One `\newcommand` per axis plus a colour command. Example for "Conservative Party":

```latex
\newcommand{\ConservativePartyColor}{\#1A75BB}
\newcommand{\ConservativePartyCultural}{7.0}
\newcommand{\ConservativePartySovereignty}{6.0}
\newcommand{\ConservativePartyMilitary}{8.0}
\newcommand{\ConservativePartyEconomic}{3.0}
\newcommand{\ConservativePartyClass}{1.0}
\newcommand{\ConservativePartyLiberty}{2.0}
```

Command names are derived from the actor display name by removing non-alphanumeric characters.

## Including Artifacts in a LaTeX Document

### Inline Scores

```latex
\input{paper-artifacts/actors/Conservative-Party.tex}
The Conservatives score \ConservativePartyCultural{} on the Cultural axis.
```

### Radar Chart Figure

Option A — `svg` package (requires Inkscape on PATH):

```latex
\usepackage{svg}
...
\begin{figure}[h]
  \centering
  \includesvg[width=0.45\textwidth]{paper-artifacts/actors/Conservative-Party.svg}
  \caption{Six-axis profile of the Conservative Party.}
  \label{fig:conservative-party}
\end{figure}
```

Option B — PDF conversion (no Inkscape needed):

```bash
cd paper-artifacts/actors
for f in *.svg; do rsvg-convert -f pdf -o "${f%.svg}.pdf" "$f"; done
```

```latex
\usepackage{graphicx}
...
\begin{figure}[h]
  \centering
  \includegraphics[width=0.45\textwidth]{paper-artifacts/actors/Conservative-Party.pdf}
  \caption{Six-axis profile of the Conservative Party.}
\end{figure}
```

### Comparison Chart

```latex
\begin{figure}[h]
  \centering
  \includesvg[width=0.7\textwidth]{paper-artifacts/comparisons/uk-parties.svg}
  \caption{UK political parties on the six-axis compass.}
  \label{fig:uk-parties}

```

## Comparison Groups

Groups are defined in `scripts/paper-config.json`. The current groups are:

| Group ID | Title | Actors |
|----------|-------|--------|
| `uk-parties` | UK Political Parties | Conservative Party, Labour Party, Reform UK, Liberal Democrats, Green Party, SNP, Plaid Cymru |
| `us-parties` | US Political Parties | US Democrats, US Republicans |
| `economic-historical` | Economic Historical Figures | John Maynard Keynes, Milton Friedman |
| `wwii-figures` | World War II Political Figures | Clement Attlee, Winston Churchill, Adolf Hitler, Franklin D. Roosevelt, Benito Mussolini, Joseph Stalin |
| `uk-vs-us` | UK vs US Major Parties | Conservative Party, Labour Party, US Democrats, US Republicans |

To add a new group, edit `scripts/paper-config.json` and run `npm run generate-artifacts`.

## Regenerating Artifacts

After any change to actor data (`data/actors/*.json`) or chart rendering (`src/chart.js`), rebuild the artifacts:

```bash
npm run generate-artifacts
```

Then commit the updated files:

```bash
git add paper-artifacts/
git commit -m "chore(artifacts): regenerate after data update"
```

## Provenance and Traceability

Every artifact embeds:
- The tool version (from `package.json`)
- The exact Git commit SHA
- The generation timestamp
- A pointer back to the source JSON file in `data/actors/`

This means a reviewer can, at any point in the future, check out the stated commit and reproduce the exact same artifacts.
